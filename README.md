# AgroRIA

Веб-застосунок на Django для продажу вживаної агротехніки.

## Опис

AgroRIA об'єднує каталог техніки з фільтрами, сторінки деталей товару, кошик, оформлення замовлення, облікові записи користувачів та адміністративну панель.

Основні типи техніки:
- Комбайни
- Трактори
- Обприскувачі

## Можливості

- Каталог товарів з пошуком та фільтрацією за маркою і типом техніки
- Сторінка деталей товару з технічними характеристиками і галереєю фото
- Поле місцезнаходження техніки у моделях і на сторінках товарів
- Кошик і базовий процес оформлення замовлення
- Реєстрація, вхід і профіль користувача
- Кастомізована Django Admin панель

## Технології

- Python 3
- Django 6
- SQLite
- Bootstrap 5
- django-redis
- python-dotenv

## Структура проєкту

- fashion_project: налаштування Django-проєкту
- products: каталог техніки, моделі, сторінки списку та деталей
- card: логіка кошика
- order: оформлення і перегляд замовлень
- account: облікові записи користувачів
- templates: HTML-шаблони
- static: стилі та статичні ресурси
- media: завантажені зображення

## Швидкий старт

1. Клонувати репозиторій або відкрити поточну папку проєкту.
2. Створити та активувати віртуальне середовище.
3. Встановити залежності:

    pip install django django-redis python-dotenv pillow

4. Створити файл .env у корені проєкту та заповнити мінімальні змінні:

    SECRET_KEY=your_secret_key
    DEBUG=True
    REDIS_URL=redis://localhost:6379/0

5. Виконати міграції:

    python manage.py makemigrations
    python manage.py migrate

6. Створити адміністратора:

    python manage.py createsuperuser

7. Запустити сервер:

    python manage.py runserver

8. Відкрити сайт у браузері:

    http://127.0.0.1:8000/

## Основні маршрути

- Головна: /
- Каталог: /products/
- Контакти: /contacts/
- Адмін-панель: /admin/

## Дані та медіа

- База даних за замовчуванням: db.sqlite3
- Завантажені файли: папка media
- Статика під час розробки: папка static

## Примітки

- Для кешування використовується Redis, якщо доступний REDIS_URL.
- Якщо Redis недоступний, застосунок переходить на локальний in-memory кеш.
- Для зображень у моделях потрібен пакет pillow.
